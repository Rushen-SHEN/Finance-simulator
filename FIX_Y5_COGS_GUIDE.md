# 🔧 Y5 COGS 率修复指南（已部署）

## 问题

Y5 COGS 率显示 **12%** 而非预期的 **35%**（neutral 场景）

## 原因

您的浏览器本地存储含有旧的模型数据，其中缺少新添加的"制造 Overhead 乘数"字段。

## ✅ 快速修复（推荐）

### 方案 1：使用新增的"完全恢复"按钮（最简单）

1. **打开参数控制台**
   - 点击页面右上角 ⚙️ 按钮

2. **点击"⟲ 完全恢复"按钮**
   - 位置：控制台右上角（红色按钮）
   - 此时会弹出确认对话框

3. **确认警告**
   - 阅读将清除的数据说明
   - 点击"确定"按钮

4. **验证结果**
   - 页面自动刷新
   - 所有参数恢复为默认值
   - **Y5 COGS 率现在应显示 35%**

---

### 方案 2：手动清空本地存储

如果方案 1 不起作用，在浏览器控制台运行以下代码：

```javascript
localStorage.clear();
location.reload();
```

**步骤**：
1. 按 `F12` 打开开发者工具
2. 选择"控制台"选项卡
3. 复制并粘贴上述代码，按 Enter
4. 页面自动刷新
5. 所有数据将重置到默认值

## 技术细节

### 发生了什么？

| 步骤 | 状态 |
|-----|------|
| 代码部署 | ✅ Y5 COGS 计算公式已修正（包含 2.8× overhead 乘数） |
| 新参数添加 | ✅ `overhead_multiplier: 2.8` 已添加到 neutral 场景 |
| 页面刷新后 | ⚠️ 旧的本地存储被加载，缺少新字段 |
| 计算时 | ❌ 缺失的字段被视为 `undefined`，回退到 1.0× 乘数 |
| 结果 | 8% COGS（实际应该 35%） |

### 为什么会这样？

浏览器的 localStorage 缓存了在添加新功能之前保存的旧模型数据。新代码无法找到预期的 `overhead_multiplier` 字段，导致使用了错误的默认值。
